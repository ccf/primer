## Overview

The SessionEnd hook is the data collection layer of Primer. It runs automatically at the end of each Claude Code, Codex CLI, or Gemini CLI session, extracts metadata from the transcript, and uploads it to your Primer server. Zero configuration after installation -- it captures every session silently in the background.

## How It Works

When a Claude Code session ends, the hook:

1. **Reads the transcript** -- The JSONL file in `~/.claude/projects/` containing every message, tool call, and metadata block from the session.
2. **Parses structured data** -- Extracts token counts, tool usage, model names, timing, git context, and session metadata from the raw transcript.
3. **Loads facets** -- Checks `~/.claude/usage-data/facets/` for qualitative analysis data (goal, outcome, friction) if available.
4. **Uploads to the server** -- POSTs the extracted `SessionMetadata` payload to `/api/v1/ingest/session`.
5. **Fails silently** -- If the server is unreachable, the hook exits without error. Use `primer sync` to recover missed sessions later.

The entire process completes in under a second and never blocks your terminal.

## Supported Agents

Primer supports multiple AI coding tools through agent-specific extraction logic. The hook automatically detects the agent type and uses the appropriate parser.

| Agent | Format | Notes |
|-------|--------|-------|
| Claude Code | JSONL transcript | Full support including tool use blocks, per-model token counts, and cache metrics |
| Codex CLI | OpenAI-compatible format | Conversation format extraction with tool call mapping |
| Gemini CLI | Gemini conversation format | Basic session metadata extraction |

All agents upload to the same `/api/v1/ingest/session` endpoint. The `agent_type` field on each session record identifies the source, allowing you to filter and compare across tools in the dashboard.

<Callout type="tip" title="Multi-agent teams">
Teams using multiple AI coding tools get a unified view in the dashboard. Session analytics, cost tracking, and friction reports work across all agent types.
</Callout>

## What Data Is Extracted

The extractor produces a `SessionMetadata` object covering identity, timing, volume, tokens, content, and git activity.

### Identity

| Field | Source | Description |
|-------|--------|-------------|
| `session_id` | Transcript metadata | Unique identifier for the session |
| `project_path` | Session environment | Absolute path to the project directory |
| `project_name` | Derived from path | Last component of the project path |
| `git_branch` | Git environment | Current branch at session start |
| `git_remote_url` | Git config | Remote origin URL for repository matching |

### Agent

| Field | Source | Description |
|-------|--------|-------------|
| `agent_type` | Session metadata | `claude_code`, `codex_cli`, or `gemini_cli` |
| `agent_version` | Session metadata | Version string of the agent |
| `permission_mode` | Session metadata | Permission level the agent was running under |
| `end_reason` | Session metadata | Why the session ended (user exit, error, timeout) |

### Timing

| Field | Source | Description |
|-------|--------|-------------|
| `started_at` | Transcript timestamp | When the session began |
| `ended_at` | Transcript timestamp | When the session ended |
| `duration_seconds` | Calculated | Wall-clock duration of the session |

### Volume

| Field | Source | Description |
|-------|--------|-------------|
| `user_message_count` | Transcript messages | Number of user prompts |
| `assistant_message_count` | Transcript messages | Number of assistant responses |
| `total_message_count` | Transcript messages | Total messages in the conversation |
| `tool_call_count` | Tool use blocks | Total tool invocations across all tools |

### Tokens

Token counts are captured at both the session level (aggregated) and per model.

| Field | Source | Description |
|-------|--------|-------------|
| `input_tokens` | Usage fields in messages | Total tokens sent to the model |
| `output_tokens` | Usage fields in messages | Total tokens generated by the model |
| `cache_read_tokens` | Cache usage fields | Tokens served from prompt cache |
| `cache_creation_tokens` | Cache usage fields | Tokens written to prompt cache |

Per-model breakdowns are stored separately in the `model_usages` table, allowing cost calculation at the model level using Primer's pricing config.

### Content

| Field | Source | Description |
|-------|--------|-------------|
| `first_prompt` | First user message | First 500 characters of the opening prompt |
| `summary` | Session summary | Summary generated by the agent, if available |

### Git Commits

For each commit made during the session, the extractor captures:

| Field | Source | Description |
|-------|--------|-------------|
| `sha` | Git log | Full commit SHA |
| `message` | Git log | Commit message |
| `author` | Git log | Commit author name and email |
| `files_changed` | Git diff stats | Number of files modified |
| `lines_added` | Git diff stats | Lines added across all files |
| `lines_deleted` | Git diff stats | Lines deleted across all files |

Git commit data powers the [GitHub Integration](../github/) features, linking sessions to pull requests for AI contribution tracking.

## Installation

The recommended way to install the hook is through the CLI:

```bash
primer hook install
```

This requires a working Primer setup. If you haven't run `primer setup` yet, do that first -- see [Installation](../installation/).

### Manual Installation

If you prefer manual setup, set your environment variables and run the install script directly:

```bash
export PRIMER_SERVER_URL=http://your-primer-server:8000
export PRIMER_API_KEY=primer_...
python scripts/install_hook.py
```

### What Gets Written

The installer adds a `SessionEnd` hook entry to `~/.claude/settings.json`:

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "python -m primer.hook.extractor"
          }
        ]
      }
    ]
  }
}
```

The empty `matcher` means the hook runs for every session regardless of project. The `command` type runs the extractor as a subprocess after each session ends.

<Callout type="note" title="Existing hooks preserved">
The installer appends to your existing hooks configuration. Any other `SessionEnd` hooks you have configured are not affected.
</Callout>

## Verification

Check the hook is registered:

```bash
primer hook status
```

This verifies:
- The hook entry exists in `~/.claude/settings.json`.
- The Primer server is reachable.
- Your API key is valid.

After completing a Claude Code session, verify the data was uploaded:

```bash
curl http://localhost:8000/api/v1/sessions \
  -H "x-admin-key: your-admin-key" | python -m json.tool
```

You should see your session with token counts, tool usage, and timing data populated.

## Offline Resilience

If the Primer server is unreachable when a session ends, the hook exits silently without disrupting your workflow. No error messages are shown, and Claude Code behaves normally.

<Callout type="note" title="Timeout">
The hook has a 10-second timeout. If your server is slow or unreachable, the hook gives up quickly so it never blocks your terminal.
</Callout>

### Recovering Missed Sessions

Use `primer sync` to backfill any sessions that were missed due to server downtime or network issues:

```bash
primer sync
```

This scans your local Claude Code transcript directory, identifies sessions not yet on the server, and uploads them in bulk.

You can also sync from within Claude Code using the MCP sidecar:

```text
> Sync my recent sessions.
```

Claude calls the `sync` MCP tool, which performs the same backfill operation. See [MCP Sidecar](../mcp/) for setup.

## Troubleshooting

### Hook not firing

Verify the hook exists in your settings:

```bash
primer hook status
```

If it reports "not installed", re-run `primer hook install`. Also check that `python -m primer.hook.extractor` runs without import errors:

```bash
python -m primer.hook.extractor --help
```

### Sessions not appearing

If the hook fires but sessions don't appear on the server:

1. Check the server is running: `curl http://localhost:8000/health`
2. Verify your API key is valid: the key in `~/.primer/config.toml` must match an active engineer account.
3. Check ingest events for errors: `curl http://localhost:8000/api/v1/sessions -H "x-admin-key: ..."` and look for recent entries.

### Token counts are zero

Some older Claude Code versions don't include usage data in the transcript. Update to the latest version of Claude Code for full token tracking.

## Uninstalling

```bash
primer hook uninstall
```

This removes the `SessionEnd` entry from `~/.claude/settings.json`. Your existing session data on the Primer server is not affected -- only future sessions stop being captured.

To re-enable later, run `primer hook install` again.
